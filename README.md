# Set up Docker

Create the Network and the Docker Containers.
```
$ ./create-network.sh && docker-compose up
```
To give permissions to the file if needed

```
$ chmod +x create-network.sh
```

## Mobile

<details>
<summary>IDE</summary>

Configure your IntelliJ for Flutter for this project
- Settings -> Langagues & Frameworks -> Dart -> Enable Dart Support:
  - Add path to Dart SDK `$RootToFlutter/flutter/bin/cache/dart-sdk`
  - Choose the module {project_name}.apps
- Settings -> Langagues & Frameworks -> Flutter -> Enable Flutter Support:
    - Add path to Flutter SDK `$RootToFlutter/flutter`
- Install the following plugins:
  - Flutter
  - Dart
  - Flutter Intl

</details>
<details>
<summary>Icon & Splash</summary>

To generate new icon and splash screen for the app, replace the files present in `lib/assets/images/logo` and `lib/assets/images/splash`.
Some rules to keep in mind

- Icon image should be 1024x1024 (splash can be the same or similar)
- Android icon adaptive (logo_transparent) should be transparent.
- Splash image should have 4x Pixel Density (or ~640DPI)

After that, generate the icons and splash

```
$ nx pub-get mobile
$ nx icons mobile //Icons
$ nx splash mobile //Splash
```

</details>
<details>
<summary>Run</summary>

The flutter app comes configured with different flavours (dev, qa, prod).<br>
To run for each environment, use the following command:<br>

```
$ nx run mobile //PROD
$ nx run-qa mobile //QA
$ nx run-dev mobile //DEV
```

</details>
<details>
<summary>Other commands</summary>

All commands can be checked in `apps/client/mobile/project.json`

```
$ nx gen-l10n mobile //generate translations
$ nx build-runner mobile //Run build runner for freezed model generation
$ nx build-ipa mobile //Build iOS
$ nx build-appbundle mobile //Build Android
```

</details>

## Back Office

<details>
<summary>Design</summary>

In order to define the colors for the project, we must edit the values present in tailwind.config.ts present in the backoffice folder.
For this, change the values in the `customPalettes` object to the ones desired for the project.

For logo related assets, replace the following files:

- apps/client/backoffice/src replace favicon-16x16.png and favicon-32x32.png
- apps/client/backoffice/src/assets/images/logo replace both files (they are used in the splashscreen, sidemenu and other place. They can be the same file if there is no full version of a logo)

</details>
<details>
<summary>Commands</summary>

- nx serve backoffice - DEVELOPMENT
- nx build backoffice (--configuration to select environment) - BUILD

</details>

<details>
<summary>Translations (with Transloco)</summary>

- add JSON file in the `assets/i18n` folder
- Go to `client/backoffice/src/app/core/transloco/transloco.module.ts` and add the new language to the array.

</details>
